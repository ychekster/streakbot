# StreakBot

Telegram-бот для отслеживания привычек через стрики — серии непрерывного
выполнения задач. Создаёт расписание, ежедневно напоминает и считает стрик 🔥

## Технологический стек

| Компонент | Библиотека |
|---|---|
| Telegram API | `aiogram` 3.x (async, FSM) |
| ORM | `SQLAlchemy` 2.0 (async) |
| БД | `aiosqlite` (SQLite) → легко на PostgreSQL |
| Миграции | `alembic` |
| Планировщик | `APScheduler` (AsyncIOScheduler) |
| Конфиг | `python-dotenv` + `pydantic-settings` |
| Часовые пояса | `pytz` + `timezonefinder` + `geopy` |
| Картинки | `Pillow` (PNG-статистика `/stats` поверх шаблона) |
| Логи | `loguru` |

## Установка и запуск

```bash
git clone <repo-url>
cd StreakBot

python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env          # Windows: copy .env.example .env
# Заполнить .env: указать BOT_TOKEN от @BotFather

alembic upgrade head          # опционально: создать схему через миграции
python -m bot.main
```

> При запуске `python -m bot.main` таблицы создаются автоматически
> (`create_all`), поэтому шаг `alembic upgrade head` не обязателен для локального
> старта на SQLite. Alembic нужен для версионирования схемы и продакшена.

## Переменные окружения

| Переменная | Назначение | Обязательна |
|---|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather | да |
| `DATABASE_URL` | Строка подключения к БД (async-драйвер) | нет (есть дефолт SQLite) |
| `LOG_LEVEL` | Уровень логирования (`INFO` по умолчанию) | нет |
| `LOG_FILE` | Путь к файлу логов (`logs/bot.log`) | нет |

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Регистрация / главное меню |
| `/tasks` | Меню «Все задачи» / «Задачи на сегодня» → списки → карточка задачи (отметка, удаление) |
| `/today` | Список задач на сегодня с отметкой галочками (применяется сразу по нажатию) |
| `/add` | Добавить задачу |
| `/edit` | Редактировать задачу (название, частота, напоминание) |
| `/delete` | Удалить задачу |
| `/stats` | Статистика и стрики — альбом PNG-картинок (по 2 задачи на картинку) |
| `/settings` | Настройки уведомлений и пояса |
| `/help` | Список команд |
| `/cancel` | Отменить текущее действие |

Отметить выполнение можно из дайджестов и напоминаний, а также вручную —
командами `/tasks` («Задачи на сегодня») и `/today`.

## Архитектурные решения (кратко)

- **Стрик вычисляется динамически** из `TaskLog` — без хранения и
  рассинхронизации. Текущий стрик считается по последнему **окончательно закрытому**
  дню: незавершённый день не обнуляет стрик, пока не прошёл его срок восстановления
  (12:00 следующего дня по поясу пользователя). Поэтому сегодняшний и вчерашний
  (до 12:00) невыполненный день стрик не сжигают.
- **`/stats` — альбом картинок**: по каждой активной задаче рендерится PNG поверх
  готового шаблона (`services/stats_image.py`, `Pillow`) с названием, текущим и
  рекордным стриком и сеткой последних 30 дней. На одной картинке — две задачи;
  при большем числе задач отправляется альбом. При нечётном числе задач на
  последней картинке одна задача, а пустой нижний блок закрывается заглушкой.
  Дизайн (фон, логотип, подписи) — в шаблоне, код накладывает только динамические
  данные. Длинное название (блок до 450 px) переносится на вторую строку без
  уменьшения шрифта, при нехватке — обрезается многоточием. Пока картинки
  генерируются И выгружаются, виден непрерывный индикатор: сообщение «Готовлю
  статистику…» и статус «отправляет файлы» (`upload_photo` шлётся в цикле, т.к.
  живёт ~5 с); сообщение снимается только сразу после появления альбома, без
  «пустоты» между ними. Выгрузка идёт с увеличенным таймаутом, а PNG сохраняется в
  RGB с оптимизацией — это убирает таймауты, дубли и долгое ожидание.
- **Repository pattern** — весь доступ к БД через один слой.
- **DI через aiogram** — сессия, репозиторий, конфиг и планировщик
  пробрасываются в хендлеры.
- **MarkdownV2** с обязательным экранированием динамических вставок.
- **Восстановление jobs** планировщика при старте по данным из БД.
- **Баннер стриков в утреннем дайджесте**: утренний дайджест приходит как PNG-баннер
  со стриками задач (🔥 + текущий стрик + название, до 10 ячеек 2×5) поверх готового
  шаблона, с текстом дайджеста в **подписи (caption)** — одним сообщением. Баннер
  генерируется **заранее**, отдельной cron-джобой за 5 минут до дайджеста
  (`prepare_morning_banner` → временный файл), чтобы к моменту отправки он был готов;
  после отправки временный файл удаляется. Если файл не готов (например, после
  рестарта) — баннер генерируется на лету; если активных задач нет или фото не ушло —
  дайджест уходит обычным текстом. При взаимодействии с инлайн-кнопками **изображение
  не меняется** — флоу просроченных задач редактирует только подпись. Тот же шрифт
  Inter Bold и инфраструктура Pillow, что и в `/stats` (`services/morning_image.py`).
- **Дайджесты интерактивны**:
  - **Утренний**: синяя кнопка «Отметить выполненные» присылает НОВОЕ отдельное
    сообщение, идентичное ответу на команду `/today` (галочки с автосохранением).
    Само сообщение дайджеста при этом не изменяется, и от пользователя в чат ничего
    не отправляется (`today_open`). Ниже — красная кнопка
    «Отметить вчерашние задачи» (`md_*`, до 12:00, редактирует то же сообщение): выбор → «Сохранить» →
    подтверждение → «Подтвердить»; после прохождения этого флоу дайджест показывается
    без блока просроченных, и кнопка просроченных больше не возвращается (независимо
    от дальнейших действий) — логика определения просроченных при этом не меняется.
    На экране «время вышло» (после 12:00) кнопка «Подтвердить» синяя. Если
    просроченных задач нет вовсе, утренний дайджест **сразу** приходит в этом
    финальном виде (заголовок + задачи на сегодня + синяя кнопка отметки, без блока
    и кнопки просроченных).
  - **Вечерний**: текст «Итоги дня» со списком задач на сегодня (структура — как у
    утреннего без блока просроченных, отличается только заголовок). Синяя кнопка
    «Отметить выполненные» открывает `/today`-вид с автосохранением **редактированием
    того же сообщения** (флоу `dm_*`, origin `e`); последний ряд — красная «‹ Назад» к
    вечернему дайджесту. «Готово» и подтверждения нет.
- **Напоминание о задаче** — жирный заголовок «Напоминание» и строка «Пора
  выполнить задачу «…»» с **стандартной кнопкой-галочкой отметки** (как в `/today`):
  синяя ⬜ с названием задачи, по нажатию — зелёная ☑️ (и обратно). Кнопка не
  пропадает, текст не меняется; если задача уже отмечена выполненной — кнопка
  приходит сразу зелёной.
- **Дайджесты деликатны**: откладываются, если пользователь активен (<5 мин,
  максимум 3 отложки), сбрасывают состояние/клавиатуру и не отправляются дважды
  за день (смена настроек не переотправляет).
- **Уникальные имена задач**: дубликаты (без учёта регистра) запрещены.
- **«‹ Назад» при добавлении задачи**: на каждом шаге после ввода названия
  (кроме первого — выбора частоты) можно вернуться к предыдущему шагу и изменить
  выбор.
- **Редактирование задачи** (`/edit`): карточка с правкой названия, частоты и
  напоминания через редактирование одного сообщения; стрик при этом не трогается.
- **Просмотр и управление задачами** (`/tasks`): одно сообщение — меню «Все
  задачи» / «Задачи на сегодня» → список с кнопками задач (стрелочная пагинация) →
  карточка задачи (параметры, стрик, кнопка-галочка «Выполнено» (☑️/⬜, цветная)
  для сегодняшних, удаление с подтверждением). Контекст (раздел и страница)
  хранится в callback-data, на каждом шаге есть «‹ Назад».
- **Отметка списком** (`/today`): сообщение с двумя блоками (невыполненные —
  обычным текстом, выполненные — зачёркнутым) и кнопками задач с галочкой (☑️/⬜)
  и цветом (синяя — не выполнена, зелёная — выполнена) сразу, без промежуточных
  кнопок «Отметить»/«Сохранить»; нажатие на кнопку тут же фиксирует статус в БД и
  перемещает задачу между блоками. В списках задач обоих дайджестов и `/today`
  перед каждым названием — маркер «• » (команда `/tasks` его не использует).
- **Кнопки отметки — галочка + цвет**: везде, где задачи отмечают галочками
  (`/today`, оба дайджеста, карточка `/tasks`, напоминание о задаче), статус показан
  двумя способами сразу — эмодзи-галочкой в тексте (☑️ выполнено / ⬜ нет) и цветом кнопки
  (`InlineKeyboardButton.style`, Bot API 9.4): невыполненная — синяя (`primary`),
  выполненная — зелёная (`success`). Кнопки дайджестов тоже цветные: «Отметить
  выполненные» (в утреннем и вечернем) — синяя, «Отметить вчерашние задачи»/
  «Сохранить»/«‹ Назад» в флоу просроченных — красные (`danger`), «Подтвердить» —
  зелёная. На старых клиентах без поддержки 9.4 цвет дефолтный, но галочка остаётся.
- **Бесконфликтная отметка**: одну задачу можно отмечать из разных мест
  (напоминание, `/today`, дайджесты, карточка `/tasks`). Если статус успели изменить
  в одном месте, нажатие на «устаревшую» кнопку в другом не выдаёт ошибку, а тихо
  обновляет клавиатуру до актуального состояния из БД. Отметка применяется
  относительно того, что показывает кнопка (нажал ⬜ → станет ☑️), поэтому свежая
  кнопка переключает статус, а устаревшая просто синхронизируется. Подробнее — в
  [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
- **`/cancel` полностью обнуляет**: чистит FSM-состояние и убирает
  reply-клавиатуру.

Подробнее — в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
