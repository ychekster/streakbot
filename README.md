# StreakBot

Telegram-бот для отслеживания привычек через стрики — серии непрерывного
выполнения задач. Создаёт расписание, ежедневно напоминает и считает стрик 🔥

## Технологический стек

| Компонент | Библиотека |
|---|---|
| Telegram API | `aiogram` 3.x (async, FSM) |
| ORM | `SQLAlchemy` 2.0 (async) |
| БД | `aiosqlite` (SQLite) → легко на PostgreSQL |
| Миграции | `alembic` |
| Планировщик | `APScheduler` (AsyncIOScheduler) |
| Конфиг | `python-dotenv` + `pydantic-settings` |
| Часовые пояса | `pytz` + `timezonefinder` + `geopy` |
| Картинки | `Pillow` (PNG-статистика `/stats` поверх шаблона) |
| Логи | `loguru` |

## Установка и запуск

```bash
git clone <repo-url>
cd StreakBot

python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env          # Windows: copy .env.example .env
# Заполнить .env: указать BOT_TOKEN от @BotFather

alembic upgrade head          # опционально: создать схему через миграции
python -m bot.main
```

> При запуске `python -m bot.main` таблицы создаются автоматически
> (`create_all`), поэтому шаг `alembic upgrade head` не обязателен для локального
> старта на SQLite. Alembic нужен для версионирования схемы и продакшена.

## Переменные окружения

| Переменная | Назначение | Обязательна |
|---|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather | да |
| `DATABASE_URL` | Строка подключения к БД (async-драйвер) | нет (есть дефолт SQLite) |
| `LOG_LEVEL` | Уровень логирования (`INFO` по умолчанию) | нет |
| `LOG_FILE` | Путь к файлу логов (`logs/bot.log`) | нет |

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Регистрация / главное меню |
| `/tasks` | Меню «Все задачи» / «Задачи на сегодня» → списки → карточка задачи (отметка, удаление) |
| `/today` | Список задач на сегодня с отметкой галочками (применяется сразу по нажатию) |
| `/add` | Добавить задачу |
| `/edit` | Редактировать задачу (название, частота, напоминание) |
| `/delete` | Удалить задачу |
| `/stats` | Статистика и стрики — альбом PNG-картинок (по 2 задачи на картинку) |
| `/settings` | Настройки уведомлений и пояса |
| `/help` | Список команд |
| `/cancel` | Отменить текущее действие |

Отметить выполнение можно из дайджестов и напоминаний, а также вручную —
командами `/tasks` («Задачи на сегодня») и `/today`.

## Архитектурные решения (кратко)

- **Стрик вычисляется динамически** из `TaskLog` — без хранения и
  рассинхронизации.
- **`/stats` — альбом картинок**: по каждой активной задаче рендерится PNG поверх
  готового шаблона (`services/stats_image.py`, `Pillow`) с названием, текущим и
  рекордным стриком и сеткой последних 30 дней. На одной картинке — две задачи;
  при большем числе задач отправляется альбом, нечётная последняя задача рисуется
  одна (нижний блок пустой). Дизайн (фон, логотип, подписи) — в шаблоне, код
  накладывает только динамические данные.
- **Repository pattern** — весь доступ к БД через один слой.
- **DI через aiogram** — сессия, репозиторий, конфиг и планировщик
  пробрасываются в хендлеры.
- **MarkdownV2** с обязательным экранированием динамических вставок.
- **Восстановление jobs** планировщика при старте по данным из БД.
- **Дайджесты интерактивны**: оба несут inline-кнопку отметки сегодняшних задач
  (общий флоу `tm_*`) — на экране выбора видны все задачи дня кнопками с галочкой
  (☑️/⬜) и цветом (синяя — не выполнена, зелёная — выполнена); статус можно
  ставить и снимать, «Подтвердить» фиксирует новый статус всех задач. Кнопка остаётся и после
  подтверждения (текст «Отменить выполнение», если все выполнены). Утренний
  дайджест дополнительно несёт кнопку отметки вчерашних просроченных задач
  (`md_*`, до 12:00) — ниже кнопки сегодняшних. Всё взаимодействие — через
  редактирование одного сообщения.
- **Напоминание о задаче** несёт кнопку «Выполнена» (активна до 12:00 следующего
  дня).
- **Дайджесты деликатны**: откладываются, если пользователь активен (<5 мин,
  максимум 3 отложки), сбрасывают состояние/клавиатуру и не отправляются дважды
  за день (смена настроек не переотправляет).
- **Уникальные имена задач**: дубликаты (без учёта регистра) запрещены.
- **«‹ Назад» при добавлении задачи**: на каждом шаге после ввода названия
  (кроме первого — выбора частоты) можно вернуться к предыдущему шагу и изменить
  выбор.
- **Редактирование задачи** (`/edit`): карточка с правкой названия, частоты и
  напоминания через редактирование одного сообщения; стрик при этом не трогается.
- **Просмотр и управление задачами** (`/tasks`): одно сообщение — меню «Все
  задачи» / «Задачи на сегодня» → список с кнопками задач (стрелочная пагинация) →
  карточка задачи (параметры, стрик, кнопка-галочка «Выполнено» (☑️/⬜, цветная)
  для сегодняшних, удаление с подтверждением). Контекст (раздел и страница)
  хранится в callback-data, на каждом шаге есть «‹ Назад».
- **Отметка списком** (`/today`): сообщение с двумя блоками (невыполненные —
  обычным текстом, выполненные — зачёркнутым) и кнопками задач с галочкой (☑️/⬜)
  и цветом (синяя — не выполнена, зелёная — выполнена) сразу, без промежуточных
  кнопок «Отметить»/«Сохранить»; нажатие на кнопку тут же фиксирует статус в БД и
  перемещает задачу между блоками.
- **Кнопки отметки — галочка + цвет**: везде, где задачи отмечают галочками
  (`/today`, оба дайджеста, карточка `/tasks`), статус показан двумя способами
  сразу — эмодзи-галочкой в тексте (☑️ выполнено / ⬜ нет) и цветом кнопки
  (`InlineKeyboardButton.style`, Bot API 9.4): невыполненная — синяя (`primary`),
  выполненная — зелёная (`success`). На старых клиентах без поддержки 9.4 цвет
  дефолтный, но галочка остаётся.
- **`/cancel` полностью обнуляет**: чистит FSM-состояние и убирает
  reply-клавиатуру.

Подробнее — в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
