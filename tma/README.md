# StreakBot — Telegram Mini App

Мини-приложение к боту [StreakBot](../README.md): экран **«Привычки»** с годовой
сеткой выполнения и отметкой задач в один тап, в стиле iOS 26 (Liquid Glass).

Состоит из двух частей:

- **`backend/`** — API-сервер (FastAPI), который переиспользует модели и
  репозиторий бота и работает с **той же базой данных**;
- **`frontend/`** — клиент (React + TypeScript + Vite), открывается как Telegram
  Web App.

│   ├── auth.py          # верификация Telegram initData (HMAC-SHA256)
│   ├── database.py      # подключение к общей БД бота
│   ├── dependencies.py  # DI: настройки, сессия+репозиторий, текущий пользователь
│   ├── schemas.py       # схемы ответов API (контракт с фронтендом)

## Backend (API)

### Технологии

FastAPI + uvicorn (async). Доступ к данным — только через репозиторий бота
(`bot.database.repository.Repository`); своя логика доступа к БД не дублируется.

### Как работает

- **Общая база с ботом.** Движок и сессии строятся поверх того же `DATABASE_URL`
  из общего `.env` — никакой второй БД. Таблицы создаёт бот; API только читает и
  пишет.
- **Авторизация.** Каждый запрос обязан нести заголовок
  `Authorization: tma <initData>`. Подпись `initData` проверяется по алгоритму
  Telegram Web Apps (HMAC-SHA256 с секретом, выведенным из `BOT_TOKEN`) при
  **каждом** запросе; без валидной подписи — `401`. Дополнительно проверяется
  возраст подписи (`TMA_AUTH_TTL_SECONDS`, по умолчанию сутки; `0` — отключить).
- **Единый формат ошибок:** `{"error": {"code": "...", "message": "..."}}` с
  осмысленным HTTP-статусом.

### Эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/tasks` | Список активных привычек пользователя с историей выполнения за последние 182 дня (7×26). |
| `POST` | `/tasks/{task_id}/toggle` | Отметить / снять отметку выполнения задачи за сегодня. Возвращает обновлённую привычку. |
| `GET` | `/health` | Проверка живости сервиса. |

Формат привычки в ответе:

```jsonc
{
  "id": 12,
  "name": "Зарядка",
  "done_today": true,
  // 182 булевых значения, старое → сегодня; true — день выполнен (статус done)
  "history": [false, true, false, /* ... */ true]
}
```

«Сегодня» вычисляется в часовом поясе пользователя (как в боте); индекс 0 в
`history` — самый старый день, последний элемент — сегодня.

### Запуск

> Запускать **из корня репозитория** — backend импортирует пакет `bot`, а
> относительный путь SQLite в `DATABASE_URL` отсчитывается от рабочей директории.

```bash
# из корня репозитория, в том же виртуальном окружении, что и бот
pip install -r tma/backend/requirements.txt   # подтянет и зависимости бота

python -m tma.backend.main
# или с автоперезагрузкой при разработке:
uvicorn tma.backend.main:app --reload --port 8000
```

### Переменные окружения (из общего `.env`)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | Токен бота — для проверки подписи `initData` | — (обязательна) |
| `DATABASE_URL` | Та же строка подключения, что у бота | SQLite-файл бота |
| `TMA_HOST` | Хост API-сервера | `0.0.0.0` |
| `TMA_PORT` | Порт API-сервера | `8000` |
| `TMA_ALLOWED_ORIGINS` | Разрешённые источники CORS (через запятую; `*` — любой) | `*` |
| `TMA_AUTH_TTL_SECONDS` | Максимальный возраст `initData` в секундах (`0` — без проверки) | `86400` |

## Frontend (TMA)

### Технологии

React + TypeScript + Vite, стилизация — **CSS Modules** + CSS-переменные
(дизайн-токены). Никаких UI-китов и компонентных библиотек — только нативный
браузерный рендеринг.

### Поведение

- При открытии вызывает `WebApp.ready()` и `WebApp.expand()` — приложение во весь
  экран, без верхнего зазора; фон/шапка красятся под дизайн-токен фона.
- Авторизация — автоматически через `initData` из `window.Telegram.WebApp`,
  которая уходит в заголовке каждого запроса.
- На время загрузки показывается скелетон; ошибки сети/API, пустой список и запуск
  вне Telegram отображаются понятными сообщениями.
- Отметка выполнения — оптимистичная: состояние меняется мгновенно, при ошибке
  запроса откатывается; на нажатие даётся тактильный отклик.

### Экран «Привычки»

Крупный жирный заголовок «Привычки» слева сверху и прокручиваемый список карточек.
Каждая карточка: строка с названием и круглой кнопкой отметки за сегодня
(незаполненный кружок ↔ заполненный с галочкой) и сетка 7×26 —
закрашенный кружок означает выполненный день (статус `done`), полупрозрачный —
пропуск/нет данных. Самый старый день — левый верхний угол, сегодня — правый
нижний.

### Запуск

```bash
cd tma/frontend
npm install

cp .env.example .env          # Windows: copy .env.example .env
# Указать VITE_API_BASE_URL — адрес API-сервера

npm run dev                   # разработка (http://localhost:5173)
npm run build                 # продакшен-сборка в dist/
npm run preview               # предпросмотр собранного
```

### Переменные окружения (на этапе сборки Vite)

| Переменная | Назначение | Пример |
|---|---|---|
| `VITE_API_BASE_URL` | Базовый URL API-сервера TMA | `http://localhost:8000` |

## Подключение к боту

Бот добавляет в главное меню (`/start`, `/help`) inline-кнопку
**«📱 Открыть приложение»** с `web_app`-URL, **только если** в `.env` задан
`TMA_URL`. Если не задан — меню работает как раньше, без кнопки.

`TMA_URL` — это публичный **HTTPS**-адрес собранного фронтенда (требование
Telegram для Web App). Локально его удобно получить туннелем (например, через
`cloudflared`/`ngrok`) к `npm run dev` или к раздаче `dist/`.

### Общий порядок запуска

1. Заполнить корневой `.env` (`BOT_TOKEN`, при необходимости `DATABASE_URL`,
   `TMA_URL`, `TMA_HOST`, `TMA_PORT`).
2. Запустить API: `python -m tma.backend.main` (из корня репозитория).
3. Собрать/поднять фронтенд и сделать его доступным по HTTPS (`TMA_URL`).
4. Запустить (или перезапустить) бота: `python -m bot.main`.

> **SQLite и конкурентный доступ.** Бот и API пишут в один файл SQLite — для
> локальной разработки этого достаточно. Для продакшена переходите на PostgreSQL
> (см. [docs/SCALING.md](../docs/SCALING.md)): меняется только `DATABASE_URL`.
